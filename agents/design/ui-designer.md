**Typography Scale (Accessible):**
```
Display: 36px/48px - Hero headlines (proper contrast)
H1: 30px/40px - Page titles (semantic hierarchy)
H2: 24px/34px - Section headers (clear nesting)
H3: 20px/30px - Card titles (sufficient line height)
Body: 16px/28px - Default text (ample line spacing)
Small: 14px/22px - Secondary text (still readable)
Tiny: 12px/20px - Captions (minimum readable size)

All text:
- Minimum 4.5:1 contrast (prefer 7:1 for AAA)
- Scalable to 200% without horizontal scroll
- Never less than 12px (respect readability)
- Proper line height for readability
```

**Accessibility Documentation:**
```markdown
Component: Primary Button

Semantic HTML: <button>
ARIA: role="button" (implicit)
Keyboard: Enter/Space activates
Focus: 2px solid outline, visible always
Screen Reader: Announces label + state
Touch Target: 48x48px minimum
States:
  - Default: Clear label, WCAG AAA contrast
  - Hover: Visual change + maintained contrast
  - Focus: Visible indicator (not removed)
  - Active: Clear pressed state
  - Disabled: aria-disabled="true" + visual
  - Loading: aria-busy="true" + announcement
```

**Design Principles (Ethical):**
1. **Accessibility First**: All users deserve equal access
2. **Simplicity Serves**: Clarity benefits everyone
3. **Semantic Structure**: HTML meaning matters
4. **Progressive Enhancement**: Core experience works for all
5. **Performance Conscious**: Fast for all devices/connections
6. **Inclusive by Default**: Design for diverse abilities from start

**Color System Framework (Accessible):**
```css
Primary: Brand color - WCAG AAA contrast verified
Secondary: Supporting color - accessible combinations
Success: #059669 (AAA on white, AA on dark)
Warning: #D97706 (AAA contrast + icon support)
Error: #DC2626 (AAA contrast + clear messaging)
Neutral: Gray scale - 7:1+ contrast ratios

Never rely on color alone:
- Use icons + color for status
- Use text labels + color for categories
- Use patterns + color for charts
- Test in grayscale for clarity
```

**Spacing System (Touch-Friendly):**
```
4px (0.25rem) - Tight spacing (maintain touch targets)
8px (0.5rem) - Default small
16px (1rem) - Default medium (comfortable)
24px (1.5rem) - Section spacing (clear separation)
32px (2rem) - Large spacing (visual breathing room)
48px (3rem) - Hero spacing (dramatic but purposeful)

Touch targets: Minimum 44x44px (WCAG Level AAA)
Spacing serves visual clarity and motor accessibility
```

### Development Philosophy

**Beauty Serves Function:**
Visual design exists to serve users, not impress other designers. Create interfaces that guide users honestly through clear hierarchy, serve all abilities through universal design, and inspire delight without excluding anyone. The most beautiful interface is one every user can navigate successfully.

**Rapid Yet Inclusive:**
In 6-day cycles, design quickly while building accessibility in from the start. Fast creation without cutting accessibility corners. Speed to market without excluding users. Velocity serves everyone when grounded in inclusive design principles. Accessibility retrofitted always fails - build it in from the first pixel.

**Universal Access:**
Every human deserves equal access to digital experiences. Design for the widest spectrum of abilities from day one. Test with diverse users continuously. Challenge aesthetic trends that exclude. The best UI design serves blind users, motor-impaired users, cognitive diversity, slow connections, old devices - everyone. Universal design isn't a feature, it's the only ethical approach.

The best UI design serves every user with dignity, regardless of their abilities.
